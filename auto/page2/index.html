About me

My name is Fernando Espinosa, and I'm a senior ruby developer based in Madrid, Spain. I've been doing software development for over 12 years. I really love it, and attention to detail and quality of code are my main concerns.

After several years working for a big consultancy company, a few more running my own company and doing freelance work, I currently work with the awesome team of Vizzuality.

In my portfolio you can check a selection of my web projects. If you want to see more, you can take a look to these too: AFG, Methane Gas Hydrates, Map My Data, Otro Bache and Protected Planet. All of them, with my friends of Vizzuality.

Besides the web projects I've created a couple of ruby gems. The first one is a client for CartoDB, a geospatial database in the cloud. The second one is a RubyMotion wrapper for the OCR engine Tesseract.

I've also created some products in my spare time just by myself. The last one is the Ikeameter, a tool to help people choose Ikea furnitures using the measures of the gap they want to furnish. Currently, I'm working in the iOS app, entirely developed using RubyMotion.

About my skills, I can perform without problems both front-end and back-end development, I love both of them and my knowledge of each of them are comparable. My object oriented background is quite big, as well as my experience working with relational databases (Oracle, Mysql, POstgreSQL, DB2...). Unix administration tasks aren't a problem either.

In short, so many years in software development is long enough for much. If you want to know my experience with a certain technology, don't hesitate, ask me.

Oh! I almost forget: I'm a vim enthusiast, and I use it in a daily basis to code, browse the web, to manage to-do lists, to write texts, etc. You can check my vim setup and other stuff in my github account.

If you want further info about my career, you can check my linkedin profile.

Hire me

If you need a ruby back-end developer, a front-end guy or an iOS programmer, I may be able to help you.

First of all, I want to make it clear I do not work for projects with fixed scope/budget, because I don't believe in them. I can give you a rough estimate of what a project could cost, but I can't compromise myself with a date because almost certainly I won't be able to fulfill it

I do iterative development, and I *do* compromise myself to a complete dedication to the project, to daily deploys, to attention to detail and, in general, to deliver a final result that satisfies both client and me.

There're two ways to work with me:

- for a fixed ammount of time. As the name says, I'll work with you for a fixed ammount of time, in which I'll perform the previously planned tasks for each week. In this case, I do weekly charges, for as long as the client wants to keep working with me.

- by project. In this case, a series of features will be defined for the project, and a first delivery date will be scheduled (without fixed scope). Once the first delivery is made, if any features still remains and the client wants to continue working in the project, a new date will be scheduled, so on so forth.

In both cases, payments will be made weekly. My daily rate, depending on how complex the tasks are and how long the contract is, is 800$/day for full availability, and 500$/day for partial availability.

Although initially my services doesn't include any kind of visual or ux design, and it's preferable that you count on it previously, you can also hire me for this task if you want. But I warn you: this isn't my speciality, so don't expect the same quality you'll expect from a specialized designer. If you want to count on me for this anyway, my daily rate is the same I've mentioned previously.

For new projects, you must know that for the majority of them it's imposible to deliver anything of enough quality for development periods below of 6-8 weeks with full availability. This supposes a minimal budget of 24,000$-32,000$. Although it will always depend on the project, below this threshold is really hard to garantee the quality I like to achieve, therefore if I think the development period is too tight you may not be able to count on me for the project.

Of course, as I said, this only applies for new projects. If you need me to support a development team, to perform maintenance on an project, or to improve an existing codebase in bad shape, you can count on me for a minimum period of two weeks.

As for my methodology, I always make tdd over the features to be developed, and I like to manage the project with tools like "Pivotal Tracker", since they help me to manage both the pace of the project and the client's expectatives.
